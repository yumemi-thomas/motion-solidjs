# LazyMotion

<Todo>
  solidjs-motion doesn't provide `LazyMotion` / lazy feature bundles (deferred to v0.3+). Use the
  `motion` proxy directly.
</Todo>

`LazyMotion` provides animation features to the [`m`](/docs/m) component. Wrap a subtree and pass a
feature bundle; `m.*` components inside that subtree load only those capabilities.

```tsx
import { LazyMotion, m, domAnimation } from 'solidjs-motion'

function App() {
  return (
    <LazyMotion features={domAnimation}>
      <m.div animate={{ x: 100 }} />
    </LazyMotion>
  )
}
```

## Feature bundles

| Bundle         | Includes                                                     |
| -------------- | ------------------------------------------------------------ |
| `domAnimation` | `animate`, `hover/Tap/Focus`, variants, exit                 |
| `domMax`       | Everything in `domAnimation` plus `drag`, `layout`, `inView` |
| `domMin`       | Bare minimum: initial to animate only                        |

Pass a function to load lazily:

```tsx
<LazyMotion features={() => import('./animations').then((m) => m.default)}>
  {/* children */}
</LazyMotion>
```

## Props

- `features` - feature bundle or async factory
- `strict` - if `true`, using `motion.*` instead of `m.*` throws
